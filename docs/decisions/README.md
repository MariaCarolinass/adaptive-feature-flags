# Architectural Decision Records (ADR)

Esta pasta registra decisões técnicas relevantes, contexto e tradeoffs.

## Como usar

1. Crie um novo arquivo com prefixo numérico: `000X-titulo-curto.md`.
2. Use o template em `adr-template.md`.
3. Não sobrescreva histórico: se a decisão mudar, crie um novo ADR.
4. Referencie PR/issue/commit quando existir.
